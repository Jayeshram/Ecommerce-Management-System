\# E-Commerce Management System



\## Project Overview



A comprehensive, production-style relational database engineered in MySQL to power a full-scale e-commerce platform. This project models the complete customer-to-fulfillment lifecycle — from product discovery and cart management to order processing, payment reconciliation, shipment tracking, and post-purchase reviews — across a robust, fully normalized 12-table schema with multiple interconnected records.



\## Database Concepts Demonstrated



\* Schema Design: Primary Keys, Foreign Keys, Unique Constraints, Check Constraints

\* Data Aggregation: COUNT, SUM, AVG, MAX, MIN

\* Data Grouping \& Filtering: GROUP BY, HAVING

\* Relational Querying: INNER JOIN, LEFT JOIN (multi-table joins)

\* Advanced Querying: Correlated and Non-Correlated Subqueries

\* Analytical Functions: Window Functions — ROW\_NUMBER(), RANK(), DENSE\_RANK() with PARTITION BY

\* Reusable Logic: Views for simplified data access

\* Procedural Programming: Stored Procedures with parameterized inputs

\* Automation \& Data Integrity: Triggers for real-time inventory updates and business rule enforcement



\## Database Architecture



\### 12 Fully Normalized Tables



1\. Categories – Product categorization

2\. Sellers – Vendor/marketplace management

3\. Customers – Customer profiles

4\. Products – Product catalog

5\. Inventory – Real-time stock tracking

6\. Cart – Active shopping sessions

7\. Orders – Order lifecycle management

8\. Order\_Items – Line-item order details

9\. Payments – Transaction processing

10\. Shipments – Logistics and delivery tracking

11\. Reviews – Customer feedback and ratings

12\. Return\_Requests – Post-sale returns management



\## Key Features



\* End-to-end order processing pipeline (Cart → Order → Payment → Shipment → Review)

\* Automated inventory management using triggers

\* Data integrity validation for business rules

\* Revenue and performance analytics using aggregate functions and window functions

\* Reusable reporting views for simplified data access

\* Parameterized stored procedures for dynamic querying
